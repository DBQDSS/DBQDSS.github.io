贪心算法

贪心算法的基本思想是：每次选择最短的任务进行分配，直到所有任务都分配完毕。

例3：排序算法的效率

算法 最坏情况下 平均情况下
插入排序 O(n²) O(n²)
冒泡排序 O(n²) O(n²)

默认是从小到大排序。

插入排序

  • 排序过程: 对于每个元素，它与它前面的元素依次比较，并将它插入到合适的位置。（向前比对，往前插）

冒泡排序

  • 排序过程: 对于每个元素，它与它后面的元素依次比较，并将它交换到合适的位置。（向后比对，往后冒）

快速排序

  • 排序过程: 选取一个元素作为基准，将所有小于基准的元素放到左边，所有大于基准的元素放到右边，然后递归地排序左右两边。

二分归并排序

  • 二分归并排序的过程: 先将数组分成两半，分别排序，然后将两个有序数组合并成一个有序数组。

  • 这样的问题有数千个, 大量存在于各个应用领域。
  • 至今没找到有效算法: 现有的算法的运行时间是输入规模的指数或更高阶函数。

定理1

定理：设 fff 和 ggg 是定义域为自然数集合的函数。

  1. 如果 lim⁡n→∞f(n)g(n)\lim_{n \to \infty} \frac{f(n)}{g(n)}limn→∞​g(n)f(n)​ 存在，并且等于某个常数 c>0c > 0c>0，那么 f(n)=Θ(g(n))f(n) = \Theta(g(n))f(n)=Θ(g(n))。
  2. 如果 lim⁡n→∞f(n)g(n)=0\lim_{n \to \infty} \frac{f(n)}{g(n)} = 0limn→∞​g(n)f(n)​=0，那么 f(n)=o(g(n))f(n) = o(g(n))f(n)=o(g(n))。

定理2

定理：设函数 f、g、h 的定义域为自然数集合，

  1. 如果 f=O(g)f=O(g)f=O(g) 且 g=O(h)g=O(h)g=O(h)，那么 f=O(h)f=O(h)f=O(h)。
  2. 如果 f=Ω(g)f=\Omega(g)f=Ω(g) 且 g=Ω(h)g=\Omega(h)g=Ω(h)，那么 f=Ω(h)f=\Omega(h)f=Ω(h)。
  3. 如果 f=o(g)f=o(g)f=o(g) 且 g=o(h)g=o(h)g=o(h)，那么 f=o(h)f=o(h)f=o(h)。

函数的阶之间的关系具有传递性。

定理3

定理：假设函数 fff 和 ggg 的定义域为自然数集合, 若对某个函数 hhh, 有 f=O(h)f=O(h)f=O(h) 和 g=O(h)g=O(h)g=O(h)，那么 f+g=O(h)f+g=O(h)f+g=O(h)。
